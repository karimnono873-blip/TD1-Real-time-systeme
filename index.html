Correction Série 03 : Ordonnancement Temps Réel

Cette page contient la résolution des exercices de la Série 03 [cite: 1] utilisant les notations classiques (C: temps d'exécution, P: période, D: deadline, N: nombre de tâches)[cite: 2].

Exercice 1 [cite: 3]

Jeu de tâches : T1(P=50, C=10), T2(P=40, C=10), T3(P=30, C=15)[cite: 7].

1. Critère de Liu et Layland [cite: 8, 9]

Le test de Liu et Layland pour RM (Rate Monotonic) stipule que l'ordonnancement est garanti si le facteur d'utilisation U est inférieur ou égal à N(21/N - 1)[cite: 4, 5].

Calculons U :
U = (10/50) + (10/40) + (15/30) = 0.2 + 0.25 + 0.5 = 0.95.

Pour N = 3, la borne est 3(21/3 - 1) ≈ 0.779.

Conclusion : Puisque 0.95 > 0.779, le critère de Liu et Layland n'est pas respecté. On ne peut pas garantir mathématiquement que le système est ordonnançable avec RM via ce simple test, il faut faire une analyse exacte (diagramme de Gantt).

2. Diagramme d'ordonnancement RM [cite: 10]

En RM, la priorité est donnée à la tâche ayant la plus petite période. Priorités : T3 > T2 > T1.

  • t=0 : T3, T2, T1 sont prêtes. T3 s'exécute (15 unités).
  • t=15 : T3 termine. T2 s'exécute (10 unités).
  • t=25 : T2 termine. T1 s'exécute.
  • t=30 : T3 est de nouveau prête. T3 préempte T1 (qui a exécuté 5 unités sur 10). T3 s'exécute.
  • t=40 : T2 est prête, mais T3 est plus prioritaire. T3 termine à t=45.
  • t=45 : T2 s'exécute jusqu'à t=50 (fin d'exécution de 5 unités).
  • t=50 : Échéance ratée pour T1 ! T1 n'a exécuté que 5 de ses 10 unités avant sa deadline à t=50.

Le jeu de tâches n'est donc pas ordonnançable avec RM.

Exercice 2 - RM et EDF [cite: 11]

Jeu de tâches : T1(P=D=9, C=3), T2(P=D=18, C=5), T3(P=D=12, C=4)[cite: 12].

1. Trace d'exécution RM (18 premières périodes) [cite: 13]

Priorités RM : T1 > T3 > T2.

Temps0-33-77-99-1212-1616-18
TâcheT1T3T2T1T3T2

À t=18, T2 n'a pu exécuter que 4 unités de temps sur les 5 requises (2 unités de 7 à 9, et 2 unités de 16 à 18). T2 rate son échéance à t=18. L'ordonnancement RM n'est donc pas faisable.

2. Trace d'exécution EDF (36 premières périodes) [cite: 14]

EDF (Earliest Deadline First) assigne la priorité dynamique à la tâche dont l'échéance absolue est la plus proche.

Un tracé minutieux sur l'hyperpériode de 36 (PPCM de 9, 12, 18) montre qu'aucune échéance n'est manquée :

  • [0-3]: T1 (D=9)
  • [3-7]: T3 (D=12)
  • [7-9]: T2 (D=18)
  • [9-12]: T1 (D=18, T2 a aussi D=18, on choisit T1)
  • [12-15]: T2 (D=18, T3 a D=24)
  • [15-18]: T3 (D=24, besoin de 4, fait 3)
  • [18-19]: T3 (D=24, termine sa dernière unité)
  • [19-22]: T1 (D=27)
  • [22-24]: T2 (D=36)
  • [24-28]: T3 (D=36)
  • [28-31]: T1 (D=36)
  • [31-34]: T2 (D=36, termine ses 3 unités restantes)
  • [34-36]: Repos (Idle)

3. Facteur d'utilisation du CPU [cite: 15]

U = (3/9) + (5/18) + (4/12) = 12/36 + 10/36 + 12/36 = 34/36 ≈ 94.4%.

Exercice 3 : RM, EDF et LLF [cite: 16]

Tâches à t=0 : T1(C=1, P=3) [cite: 17, 18], T2(C=1, P=4) [cite: 20], T3(C=2, P=6)[cite: 21].

Calcul du facteur d'utilisation (U) [cite: 23]

U = (1/3) + (1/4) + (2/6) = 4/12 + 3/12 + 4/12 = 11/12 ≈ 91.6%.

Conclusion : Comme U ≤ 1, le jeu de tâches est théoriquement ordonnançable par des algorithmes dynamiques optimaux comme EDF et LLF. Cependant, 0.916 > 0.78 (borne L&L pour n=3 [cite: 24]), on ne peut donc pas conclure directement pour RM ; un tracé est nécessaire.

Exercice 4 [cite: 25]

Système : T1(P=20, C=3, D=15), T2(P=80, C=40, D=60), T3(P=40, C=10, D=35)[cite: 26, 27].

1. Ordonnancement RMA [cite: 28]

a) Priorités : Basées sur les périodes les plus courtes[cite: 29]. Priorité : T1 > T3 > T2.

b) Facteur d'utilisation : U = 3/20 + 40/80 + 10/40 = 0.15 + 0.5 + 0.25 = 0.90. Le test d'utilisation classique suppose D = P[cite: 30]. Puisque ici D < P pour toutes les tâches, le test n'est pas applicable tel quel, et 0.90 est élevé.

2. Ordonnancement DMA (Deadline Monotonic) [cite: 32]

a) Priorités : Basées sur les deadlines relatives les plus courtes[cite: 33]. Priorité : T1 (15) > T3 (35) > T2 (60).

Note : Ici, DMA donne exactement le même ordre de priorité que RMA.

3. Ordonnancement EDF [cite: 36]

a) Facteur d'utilisation : U = 0.90[cite: 37]. Comme U ≤ 1 et que l'algorithme est EDF, si D=P le système serait valide. Puisque D < P, on doit utiliser le critère de charge processeur (Processor Demand Analysis) pour affirmer la validité[cite: 37].

2. SJF préemptif (SRTF) avec temps d'arrivée décalés [cite: 53]

Arrivées : t(P1)=0, t(P2)=1, t(P3)=2, t(P4)=3, t(P5)=4[cite: 50, 51, 52].

  • t=0: P1 s'exécute (reste 10).
  • t=1: P2 arrive (besoin 1). P2 préempte P1 (reste 9).
  • t=2: P2 termine. P3 arrive (besoin 2). P3 s'exécute.
  • t=3: P4 arrive (besoin 1). P4 préempte P3 (reste 1).
  • t=4: P4 termine. P5 arrive (besoin 5). P3 reprend (besoin 1 < P5).
  • t=5: P3 termine. P5 s'exécute (besoin 5).
  • t=10: P5 termine. P1 reprend (reste 9).
  • t=19: P1 termine.
